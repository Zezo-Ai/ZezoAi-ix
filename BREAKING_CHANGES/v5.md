# Breaking Changes v5

This document lists breaking changes introduced in Siemens Industrial Experience v5.

## Remove bootstrap font variable

Remove remaining bootstrap variable `--bs-font-sans-serif` from the styles. If you still rely on bootstrap set the font family variable via:

```css
--bs-font-sans-serif: var(--theme-font-family);
```

## Remove legacy theme class names

- Legacy theme classes such as `theme-classic-light` and `theme-classic-dark` are removed.
- Theme selection now uses HTML attributes only:
  - `data-ix-theme`
  - `data-ix-color-schema`
- Example: `<html data-ix-theme="classic" data-ix-color-schema="system">`
- IX no longer writes a default theme to the HTML element.
- A theme is applied only when both attributes are present.

### Migration rules

1. Remove legacy theme classes from `<html>` and `<body>`.
2. Set `data-ix-theme` and `data-ix-color-schema` together.
3. Do not rely on IX to inject a default theme.
4. If your app requires a default theme, define it explicitly.

### Replace

```html
<html class="theme-classic-light"></html>
```

with

```html
<html data-ix-theme="classic" data-ix-color-schema="light"></html>
```

### Notes for automated migration

- Class names in the form `theme-<theme>-<schema>` should be converted to:
  - `data-ix-theme="<theme>"`
  - `data-ix-color-schema="<schema>"`
- Remove the legacy class after extracting both values.
- If only one attribute is set, the theme is incomplete and should be treated as invalid.

### Breaking changes in `ThemeSwitcher`

| API                  | v4 behavior                           | v5 behavior                                       | Migration action                              |
| -------------------- | ------------------------------------- | ------------------------------------------------- | --------------------------------------------- |
| `getCurrentTheme()`  | Could fall back to legacy class names | Returns only `data-ix-theme`                      | Handle `undefined` when no attribute is set   |
| `setTheme()`         | Updated theme through class names     | Writes `data-ix-theme` and `data-ix-color-schema` | Pass both theme and color schema              |
| `hasVariantSuffix()` | Available                             | Removed                                           | Remove usage                                  |
| `setVariant()`       | Updated class-based variant           | Writes `data-ix-color-schema`                     | Replace variant logic with color schema logic |

### `ThemeSwitcher` before

```ts
themeSwitcher.setTheme('classic-light');
```

### `ThemeSwitcher` after

```ts
themeSwitcher.setTheme('classic', 'light');
```

### `ThemeSwitcher.modeChanged` event removed

The `modeChanged` event has been removed. Use the `themeChanged` event instead, which now includes both the configured color schema and the resolved mode.

### `ThemeSwitcher.themeChanged` event before

```ts
themeSwitcher.themeChanged.on((theme) => {
  console.log('Theme changed to', theme);
});
```

### `ThemeSwitcher.themeChanged` event after

```ts
themeSwitcher.themeChanged.on(({ theme, colorSchema, mode, isMediaChange }) => {
  console.log(
    'Theme changed to',
    theme,
    'configured schema:',
    colorSchema,
    'resolved mode:',
    mode,
    'Media change:',
    isMediaChange
  );
});
```

### Breaking changes in `ix-application` component

| API                          | v4 behavior                         | v5 behavior                                     | Migration action                       |
| ---------------------------- | ----------------------------------- | ----------------------------------------------- | -------------------------------------- |
| `theme` prop                 | Expected `IxTheme`                  | Expects the raw `data-ix-theme` value           | Pass values such as `classic`          |
| `updateTheme()`              | Used legacy theme and variant logic | Uses `ThemeSwitcher.setTheme()` with attributes | Update custom integrations accordingly |
| `themeSystemAppearance` prop | Available                           | Removed                                         | Use `colorSchema="system"` instead     |

### `ix-application` before

```tsx
<ix-application theme="classic-dark" themeSystemAppearance>
```

### `ix-application` after

```tsx
<ix-application theme="classic" colorSchema="system">
```

## Breaking changes of `ix-toast-container`

- The `toast` function now appends to the host element instead of a separate container.
- The `toast-container` class is now applied directly to the host element.

### Properties removed

- `containerClass` - The class name for the toast container is now fixed as `toast-container`.
- `containerId` - Just use `id` on the `ix-toast-container` element if needed.
